# リリースノート <a id="release-notes"></a>

## 新機能
- 取得したブロックチェーンデータをリアルタイムでKafkaにプッシュするストリーミングモジュールを実装しました
  - Kafkaクライアントを実装 (#685)
  - Kafka (#689) のオプションを追加しました
  - Kafka消費者を実装 (#697)
- リモートキャッシュの実装 (redis) (#681)
- 最新のブロックのためにRedisのpub/subを実装しました (#676)
- DBマイグレーション機能の実装 (#648)
- コールドスタート時のパフォーマンス向上のために再起動後にキャッシュを復元するStateTrieNode キャッシュダンプ機能を実装しました (#692, #702)

## 改善
- 状態トリイマイグレーションの状態APIの間違った出力を修正(#703)
- `AccountKeyLegacy` (#674) を含むロールベースのキーの検証ロジックを修正しました。

## その他
- Fetcher/Downloader/Worker を無効にするためのオプションを追加しました (#680, #688)
- 既存のチェーンデータを使用してジェネシスブロックを生成するように実装されています (#649)
- 未使用のフラグ `--cache.writethrough` (#682) を削除しました
- trie DB (#683) の読み取りに失敗したときにエラーログを追加しました
- ステーキングコントラクトのソースコードを追加しました(#699)
- 過去のブロックをテストするために `--start-block-num` を追加しました (#711)
- キーストアが作成されたときに有効性チェックを追加しました (#725)
- レベル DB (#726) の読み書きパフォーマンス指標を追加しました
