# Klaytn v1.4.0 のリリースノート <a id="release-notes-for-klaytn-v1-4-0"></a>

## New Features <a id="new-features"></a>
- Keystore V4
  - Klaytnでは新しいキーストア標準 [KIP-3](https://klaytn.github.io/kips/KIPs/kip-3) が採用され、そのアカウントをサポートしました。

## Fixes <a id='fixes'></a>
- `ステーキング情報`を更新する際のタイミングの問題によって生じる間欠的な不良ブロックの発生を修正しました。
- `debug.traceBlock` の実行中のパニックを修正しました。なぜならば、API と trie ノードのガベージコレクション間のタイミングの問題が原因だからです。
- ブロック同期中のバグレポートを修正しました。

## Improvements <a id='improvements'></a>
- ストレージの最適化
  - すべてのトランザクションではなく、すべてのブロックのストレージ ルート ハッシュを計算するようになりました。
  - Klaytn は同時に三つのハッシュルートを計算します。
- 状態試行キャッシュのサイズを増加
  - パフォーマンスを向上させるために、デフォルトのキャッシュサイズが増加しました。
- バッジャーDBバージョンをv1.6.0にアップグレードします。
- 置き換えられたメトリックパッケージ
- 追加されたパッケージの説明
- テストコードカバレッジの増加
