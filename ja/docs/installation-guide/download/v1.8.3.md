# Release Notes <a id="release-notes"></a>

Klaytn v1.8.3 では、ブロックが生成された場合やトランザクションが伝播された場合の各ノードの受信時間に応じて、トランザクションが時系列的にソートされます。 以前は、取引はガス価格で仕分けされていました。 しかし、Klaytnのガス価格は固定されているので、基本的には注文なしでした。 このバージョン変更により、KlaytnノードはFirst-Come、First-Serveベースでトランザクションを処理します。

## New Features
- 同じガス価格を持つ取引は、受信時間ごとにソートされます。 (#1282)
- 伝播中のトランザクションも受信時間の順にソートされます。 (#1309)
- ファイルに書き込まれるジャーナルトランザクションは受信時間順にソートされます。 (#1329)

## Improvements
- プリフェッチのオプションを無効にすることができ、スナップショットのメトリックが追加されました。 (#1234)
- GetCodeSize はエラーの代わりに 0 を返すようになりました。 (#1279)

## Fixes
- Worker の `LatestSignerForChainID` から `MakeSigner`へ、 `SenderCacher` を使用できるように変更されました。 (#1305)
- プレシェッチャーにパニックガードを追加。 (#1311)

## Miscellaneous
- 組み込みのソリッドテストファイルをv0.8.xにアップグレードしました。 (#1149 #1189)
- go1.18 をサポートし、CircleCI の Dockerfile を Alpine から Ubuntu 20.04 に更新しました。 (#1320)
- マイナーメンテナンス (#1265 #1270 #1273 #1308 #1306 #1312 #1313 #1314 #1322 #1327 #1327 #1338 #1349)
